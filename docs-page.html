Introdução

Sobre o seller-center

O seller center da Ozllo é uma plataforma para que sellers do segmento de moda, beleza e lifestyle consigam vender e fazer a gestão de suas mercadorias em multiplos marketplaces. Nosso seller center é integrado via API na integradora de marketplace Hub2b, que então envia os produtos para os marketplaces. Atualmente, temos 40 sellers que utilizam da nossa plataforma para cadastro de produtos e gestão de vendas.

Importante:

Nota

Essa documentação é para auxiliar os novos colaboradores a utilizar o seller-center 360.

Atenção

É importante verificar se na sua pasta main está salvo o arquivo .env, que são as variáveis de ambiente necessárias para a conexão do backend do seu projeto. Acesso exclusivo aos desenvolvedores. Entre em contato com os responsávei pelo projeto.

Configuração do ambiente

Para que o backend funcione corretamente verifique se o seu computador tem os recrusos necessário para as conexões. Após a checagem positiva, siga os passos abaixo:

Pasta seller-center-main

Depois de acessar o desktop no pront de comando, certifique-se de que a pasta seller-center-main esteja salva em seu computador, você pode adquiri-la no Github ....

npm install

Após abrir a pasta em seu pront de comando executar o comando: npm install. Assim, o computador vai instalar todos os aquivos da pasta baixada.

npm run build

Após o fim da instalação, que dependendo do compuntador pode levar até 2 minutos, executar o comando: npm run build. Server para permitir a execução de tarefas de construção e preparação necessárias para o projeto.

npm run dev

Por último basta você executar o comando: npm run dev, que o projeto vai estar funcionando, conectado ao banco de dados e com as demais plataformas que o seller-center utiliza.

APIs

Após as instalações do projeto feitas na configuração do ambiente, verifique a porta HTTP que retornou, utilizamos o endpoint https://ozllo-seller-center.herokuapp.com/. Ela é necessária para acessar os terminais de conexão. As requests feitas para essa documentação foram feitas pelo postman, mas pode ser usado outro de sua preferência.

AccountRouter

Rotas da conta do usuário, para manipular os dados da conta é preciso de autorização que é o token que retorna no email do usuário.

Criar informações pessoais para um usuário

As informações pessoais de cada usuário dependem de 2 variações, ambas utilizam o método POST, existe a opção de CNPJ e a opção de PF-pessoa física. Nessa parte é necessário um token de auth, cada login novo gera um token diferente e ele tem um tempo de duração para a segurança do usuário. Então é importante verificar se a requisição está utilizando um token válido, caso contrário a request vai dar o erro: 401-Unauthorized. O endpoint usado: https://ozllo-seller-center.herokuapp.com/account/personalInfo.

Para criar a informação de pessoa física ele exige um parametro bulean, o CPF não pode estar editado, a sintaxe deve ficar assim:

{ "_id": "985ad4a7c55f8f8w6314006",
"isPF": true,
"firstName": "Abreu",
"lastName": "Silva",
"cpf": "04489980094",
"birthday": "26-09-1985",
}

Em caso de sucesso espera-se o seguinde retorno:

{ "_id": "985ad4a7c55f8f8w6314006",
"firstName": "Abreu",
"lastName": "Silva",
"cpf": "04489980094",
"birthday": "26-09-1985",
}

Para criar um PJ os parâmetros necessário são um pouco diferente, esse formato é obrigatório a requisição vai dar erro se não estiver assim, não esqueça de verificar se o token está atualizado e por formato a inscriçãoEstadual e a inscriçãoMunicipal vem como "Isento". Diferente do CPF o CNPJ deve estar no formato editado:

{ "_id": "985ad4a7c55f8f8w6385206",
"name": "Abreu",
"cnpj": "05.858.901/0001-32",
"isPJ": true,
"razaoSocial": "Manutenção de computador",
"inscricaoEstadual": "Isento",
"inscricaoMunicipal": "Isento"
}

Caso todos os parâmetros estejam corretos o retorno é um 200 sucess e a mensagem abaixo:

{ "_id": "985ad4a7c55f8f8w6385206",
"name": "Abreu",
"cnpj": "05.858.901/0001-32",
"razaoSocial": "Manutenção de computador",
"inscricaoEstadual": "Isento",
"inscricaoMunicipal": "Isento"
}

Ciar um endereço de usuário

Para criar um endereço para o usuário use o método POST, o endpoint: https://ozllo-seller-center.herokuapp.com/account/address. O "complemento" é o único parâmetro opcional os demais são obrigatório.

{ "_id": "985ad4a7c55f8f8w6314006",
"address": "Rua São João",
"city": "Santo Antonio",
"complement": "",
"district": "Parque das Fontes",
"number": "660",
}

Caso todos os parâmetros estejam corretos o retorno é um 200 sucess e a mensagem abaixo:

{ "_id": "985ad4a7c55f8f8w6314006",
"address": "Rua São João",
"city": "Santo Antonio",
"district": "Parque das Fontes",
"number": "660"
}

Criar informações da loja para um usuário

Para criar informações da loja do usuário deve ser usado o método POST e o endpoin: https://ozllo-seller-center.herokuapp.com/account/shopInfo. Essa request é necessário colocar no Headers o Authorization - o token de login. Caso o Headersnão esteja com o token atualizado vai dar o erro 401 "Unauthorized". Após atualizar o toke, no body->raw->JSON é preciso inserir um parâmetro com o nome da loja:

{
"name": "Roupas Joana"
}

Ele vai retornar um _id, um id de usuário e o nome da loja:

{
"_id": "985ad4a7c55f8f8w6314006",
"userId": "985ad4a7c55f8f8w6314952",
"name": "Ozllo",
}

Criar o banco de informaões para o usuário

Para criar o banco de informações para o usuário dever ser usado o método POST e o endpoint vai ficar assim: https://ozllo-seller-center.herokuapp.com/accountbankInfo. Para essa request também é necessário acrescentar o Headers com o Authorization e o token de login usado. É preciso ter o parâmetro "pix" mesmo que ele seja opcional, o "account" deve ter 9 dígitos e "agency" deve ser 5 dígitos podendo ficar formatado "5632-9" caso contrário vai dar o erro 401 "Unauthorized".A sintaxe a ser executada no body->raw->JSON:

{
"bank": "004",
"account": "21546541-6",
"name": "Banco do Nordeste",
"agency": "3652-9",
"pix": ""
}

O retorno vai ser:

{
"_id": "dd87ad6a4d6a4d6ad76as",
"bank": "004",
"account": "21546541-6",
"name": "Banco do Nordeste",
"agency": "3652-9"
}

Criar um contato para um usuário

Para criar um contato para um usuário deve ser usado o método POST e o endpoin vai ficar assim: https://ozllo-seller-center.herokuapp.com/account/contact. Essa requisição também é necessário acrescentar o Headers com o Authorization e o token de login usado. A rota de contato exige um número de telefone, que deve ter 10 dígitos, e tem como opcional uma URL. A sintaxe fica assim:

{

"url": ""
}

O retorno vai ser:

{

"userId": "a4d6a4d6dad4a6d9d7daee",

AuthRouters

Essas são as rotas de autorização. As rotas de autorização precisam ter sempre o /auth no endpoint, ou elas vão retornar o erro 404 Unauthorzed.

Criar conta

Para criar uma conta utilizamos o metodo POST para enviar um novo dado com o endpoint https://ozllo-seller-center.herokuapp.com/auth/creat. A senha precisa ser forte e só aceita o email, se usar qualquer outra variável vai dar erro.

O body->raw->JSON vai ficar assim:

{ "email": "ozllo123@gmail.com",
"password": "Ozllo903$"
}

Ele vai retornar:

{ "email": "ozllo123@gmail.com",
"isActive": "false",
"role": "seller",
"id": "985ad4a6d44f8f8w63148a"
}

Ativar a conta

No email que foi utilizado para criar a conta vai ser gerado um token, a ativação manda um veryfi para o email e o token recebe a autorização pelo beckend. O metódo vai ser GET, pois estamos pegando um dado do servidor e o endpoint vai ficar desse modo então: https://ozllo-seller-center.herokuapp.com/auth/activate/:token

Verificar se o usuário pode fazer login

Para verificar se o usuário já foi ativado e pode fazer login, deve ser usado o metodo POST e o endpoint: https://ozllo-seller-center.herokuapp.com/auth/login

Fazer o login

Para fazer o login é necessário usar um metodo POST com o endpoint https://ozllo-seller-center.herokuapp.com/auth/login. Para fazer o login só é aceito o parâmetro email, se for outro vai dar erro.

O body->raw->JSON deve ficar assim:

{ "login": "ozllo123@gmail.com",
"password": "Ozllo903$"
}

Fazer logout

Para fazer o logout é necessário usar o metodo PUT com o endpoint: https://ozllo-seller-center.herokuapp.com/auth/logout. Nesse não é preciso passar um body.

Esqueci a senha

Para trocar a senha você precisa usar um metodo GET e o endpoint: https://ozllo-seller-center.herokuapp.com/auth/forgotPassword/:emaildousario. Não é necessário usar um body.

Ele vai retornar:

{
"data": "Email send to reset password"
}

Resetar a senha

Quando enviado o endpoint para forgotPassword, vai chegar no email de usuário cadastrado um token para renovar a senha o metodo que deve ser usado é GET e o endpoint vai ficar assim: https://ozllo-seller-center.herokuapp.com/auth/resetPassword/:token.

Ele vai ser verificado pelo backend para saber se o token é valido pois ele tem um perído de expiração, e se ele for válido vai retornar um id de usuário.

Depois dessa verificação deve ser usado um metodo POST e o endpoint https://ozllo-seller-center.herokuapp.com/auth/resetPassword. E o body->raw>JSON:

BankRouter

Para ver todos os bancos que estão disponíveis para serem utilizados, deve se enviado um método GET para o endpoint https://ozllo-seller-center.herokuapp.com/bank/all.

Ele retona todos os bancos com o nome e o número da agência:

{
"value": "001",
"name": "Banco do Brasil"

CategoryRouters

Essas são as rotas de categorias.

Lista de categorias

Para vizualizar a lista inteira de categorias é preciso enviar um método GET para o endpoint: https://ozllo-seller-center.herokuapp.com/category/all. Nessa requisição é necessário usar o _id do produto e o Authorization.

Ele vai retornar o código da categoria e a classe a qual ele pertence:

{
"code": "281",
"value": "Acessórios"
}

Lista de subcategorias

Para vizualizar a lista inteira de subcategorias é preciso enviar um método GET para o endpoint: https://ozllo-seller-center.herokuapp.com/category/:category_code/subcategories. Nessa requisição é necessário usar o _id do produto e o Authorization.

Ele vai retornar o código da categoria, o código e a classe a qual ele pertence:

{
"categoryCode": "281",
"code": "406",
"value": "Anabelas",
}

Lista de atributos de uma categorias

Para vizualizar a lista inteira de categorias é preciso enviar um método GET para o endpoint: https://ozllo-seller-center.herokuapp.com/category/:category_code/attributes. Nessa requisição é necessário usar o _id do produto e o Authorization.

Ele vai retornar um objeto com o código da categoria e os atributos como a cor e o tamanho:

{
"categoryCode": "281",
"attributes": [
{
"name": "size",
"type": "string",

"8"

Para excluir uma variação do produto deve ser enviado o método DELETE para endpoin com o id da variação que deseja excluir, o endpoin fica assim: https://ozllo-seller-center.herokuapp.com/product/:product_id/variation/:variation_id.Nessa requisição é necessário usar o shop_id do produto e o Authorization no Headers.

Após ser deletado ele retorna apenas: true.
